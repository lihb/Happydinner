一. 公共说明

1. 接口统一前缀为http://120.55.162.6/api/
2. 返回值统一使用json格式
3. 关于返回码（result_code字段）
	a. 返回码格式为xyyzz
	b. x表示错误分类	0-成功 1-系统级错误 2-服务级错误
	c. yy表示服务模块代码	00-通用模块	01-点餐模块	02-电影模块 03-游戏模块 04-位置
	d. zz表示具体错误代码
	e. 如10101表示发送验证码时缺失参数
	f. 0表示成功
4. 每个接口都会返回result_code字段，当接口返回失败时（即result_code!=0），还有result_desc字段作为对错误的补充描述
5. 接口公共字段（作为url参数）
	platform			int				required		平台 (1-android 2-ios)
	os_ver				string			required		操作系统版本
	device				string			required		设备信息
	app_ver				string			required		应用版本
	language			string			required		语言

二. 数据模型说明
1. MenuItemInfo              菜品信息
name						string			菜品名称
imgUrl						string			菜品图片
video						string 			菜品介绍视频
info						string			菜品介绍文字
discount					float			折扣信息
// count					int				被点的数量
// cooked					int				是否上菜，0 - 已上菜，1 - 未上菜

2. GameInfo	游戏信息
name 				string			游戏名称
imgUrl				string			游戏封面
launchInfo      	string			启动参数
packageName			string			游戏包名
downloadUrl			string			下载地址

3. MovieInfo	电影信息
name 					string			电影名称
imgUrl					string			电影海报
playtype				int				播放类型（1：网页，2：播放器）
downloadUrl				string			播放或者下载地址

4. GameType				游戏分类
typename				string					类型名称
items					jsonarray(GameInfo)		游戏列表

5. MovieType			电影分类
typename				string					类型名称
items					jsonarray(MovieInfo)	电影列表

6. MenuType				菜品分类
typename				string					类型名称
items					jsonarray(MenuItemInfo)		菜品列表


三. 接口说明

1. 根据设备号读取已下单的清单列表以及其他基本信息。
Url: /order/getorderinfo
Method: GET
Params:
	deviceid			string		required	设备号
Return:
	result_code			int			错误码: 20001-缺失参数 20002-参数格式错误
	result_desc			string		错误描述
	table_number		string		required	桌号
	xxxxxxx


2. 下单或加单，提交点菜数据
Url: /order/placeorder
Method: POST
Params:
	deviceid			string				required	设备号
	menus   			jsonarray(MenuItem)	required  	订单的菜品数据
Return:
	result_code			int			错误码: 20001-缺失参数 20002-参数格式错误
	result_desc			string		错误描述
	timestamp			string      required    下单时间

3. 发起结算，获取支付订单号
Url: /order/buyorder
Method: POST
Params:
	deviceid			string		required	设备号
Return:
	result_code			int			错误码: 20001-缺失参数 20002-参数格式错误
	result_desc			string		错误描述
	order_pay_id		string      required    支付单号

4. 获取支付结果
Url: /pay/checkresult
Method: GET
Params:
	deviceid			string			required	设备号
Return:
	result_code			int				错误码: 20001-缺失参数 20002-参数格式错误
	result_desc			string			错误描述
	state				int         	0 支付成功；1 支付失败；-1 其他

5. 获取菜品数据
Url: /menu/list
Method: Get
Params:
Return:
	result_code					int					错误码: 20001-缺失参数 20002-参数格式错误
	result_desc					string				错误描述
	menulist					jsonarray(MenuType)			菜品列表

6. 获取游戏数据
